## 4.1 변수란 무엇인가? 왜 필요한가?

---

```jsx
애플리케이션 : 데이터를 다룸
input - 데이터 처리 - output

메모리 주소에 저장된 리터럴(값)을
연산자와 피연산자, 표현식 해석(파싱)을 통해
평가(계산)
```

```jsx
값은 메모리에 저장되고 주소를 통해 접근할 수 있지만
메모리 주소를 통해 값에 직접 접근하는 것은 위험해서 막혀있음

변수 : 메모리 공간 자체 혹은 공간 식별을 위해 붙인 이름
=> 값의 위치, 메모리 주소의 주소
```

```jsx

  변수     메모리 주소  메모리
변수 이름 - 변수 주소 - 변수 값

변수에 값을 저장 : 할당(대입, 저장)
값을 읽음 : 참조
```

## 4.2 식별자

---

- 식별자 : 어떤 값을 구별해서 식별할 수 있는 고유한 이름

<aside>
💡

식별자(변수)는 값이 아니라 메모리 주소를 기억

</aside>

## 4.3 변수 선언

---

- 변수를 생성하는 것

```jsx
메모리 공간 확보, 변수 이름과 메모리 공간의 주소 연결 -> 값 저장 가능
기본적으로 undefined가 할당되어 초기화됨
```

## 4.4 변수 선언의 실행 시점과 변수 호이스팅

---

```jsx
console.log(score);

var score;
```

<aside>
💡

변수 선언은 런타임이 아니라 소스 코드 평가 과정에서 실행

⇒ 변수 호이스팅(변수 선언이 코드의 선두로 올라온 것처럼 동작)

</aside>

## 4.5 값의 할당

---

- 할당연산자 : a = b ⇒ b의 값을 a 변수에 할당
- 변수 할당 시 기존 공간을 지우고 저장하는 게 아닌 새로운 메모리 공간을 확보하고 거기에 값 저장

```jsx
불필요해진 값들은 가비지 콜렉터에 의해 메모리에서 자동 해제
```

```jsx
자바스크립트는 매니지드 언어이므로
개발자의 명시적인 메모리 할당 및 해제 불가능
```

## 4.7 식별자 네이밍 규칙

---

- 특수문자를 제외한 문자, 숫자, _, $로 이루어짐
- 숫자로 시작 x
- 예약어 제외
- 대소문자 구별
- 명확한 표현
