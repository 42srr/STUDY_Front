# 44장 REST API

- REST는 HTTP의 장점을 최대한 활용할 수 있는 아키텍쳐

## 44.1 REST API의 구성

- 자원, 행위, 표현의 3가지 요소로 구성됨
  - 자원 = URI(엔드 포인트)
  - 행위 = HTTP 요청 메서드
  - 표현 = 페이로드

## 44.2 REST API 설계 원칙

- URI는 리소스를 표현하는 것에 집중
- HTTP 요청 메서드를 통해 리소스에 대한 행위를 표현
